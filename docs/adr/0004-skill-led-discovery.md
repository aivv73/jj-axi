# Skill- and bootstrap-led discovery instead of session hooks

jj-axi uses the canonical skill for proactive agent discovery and a short bare-invocation bootstrap guide for on-demand discovery; it will not install session-start hooks. Agent hook systems have different lifecycle and configuration contracts, mutate user or project configuration, and inject state that may be stale by the time an agent acts. The skill and bootstrap teach agents when to keep using raw `jj` and when to switch to jj-axi without coupling the product to individual agent runtimes. Fresh structured repository state remains available explicitly through `inspect`.

This deliberately rejects mechanical conformance with AXI’s session-hook principle. jj-axi will document each AXI principle as applicable, adapted, or not applicable and optimize for the product’s agent UX rather than checklist completion. Environments that do not discover skills automatically receive no proactive context and must invoke the CLI explicitly to obtain the bootstrap guide.
